# おふろ魚図鑑アプリ(仮称)

3〜5歳児の「お風呂渋り」を解決する商品「おふろ魚図鑑」の相棒PWA。
21日間、氷から出てきた魚フィギュアを風呂上がりに撮影すると正体(名前・豆知識)が明かされる図鑑アプリ。

## 現在の状態: ステップ2(LINE内ブラウザでの梅tier閲覧確認)

- ルート(`/`): 照合テスト画面(ステップ1の成果物)
- `/ume/`: 梅tierの静的モック(シルエット図鑑21体+0番デモ+品質証明+LINE導線+音声テスト)

### 照合方式: MobileNetに確定(2026-07-03)

iPhone実機での比較テスト結果:

| 方式 | 同一物 | 別の物 | 結論 |
|------|--------|--------|------|
| 色ヒストグラム | (高) | 0.70〜0.88 で誤通過 | **不成立**(同じ部屋・照明だと背景の色分布が支配的) |
| MobileNet(初回テスト) | 0.70〜0.74 | 0.51〜0.57 | 採用を決定 |
| MobileNet(フィギュア再テスト・中央クロップ併用後) | **0.91〜0.96** | **0.59〜0.67** | 差0.24で**実用レベル**。しきい値0.79で運用 |

判定は「基準写真(複数枚)との最大コサイン類似度 ≧ 0.79」。撮影画像は全体と中央クロップの2通りで照合し高い方を採用(子どもの距離・フレーミングの揺れ対策)。しきい値はテスト画面のスライダーで調整できる。

本実装での補強(仕様どおり):
- 魚ごとに基準写真を複数角度で用意(マスタデータに同梱)
- 3回スコア未達で親向け手動確認ボタン

## iPhone実機での検証手順

### A. カメラ+ホーム画面追加の検証(最重要)

1. iPhoneのSafariで公開URLを開く → カメラ許可を求められたら「許可」
2. プレビューが映り「📷 撮影する」で撮影できることを確認
3. 共有ボタン → 「ホーム画面に追加」
4. ホーム画面のアイコンから起動 → 画面右上が「ホーム画面起動: ✅」になっていることを確認
5. その状態でもう一度カメラ起動・撮影できることを確認
   (iOSのバージョンによっては起動のたびにカメラ許可を聞かれる。その頻度もメモしておく)

### B. 照合精度の検証(自宅にある物でOK)

フィギュアが未完成でも、色や形が異なる小物(ぬいぐるみ・おもちゃ等)2〜3個で代用できる。

1. 「基準を登録」モードで物Aを2〜3枚撮影(角度を変えて)
2. 「判定する」モードで **同じ物A** を撮影 → スコアが高い(○)ことを確認
3. **別の物B** を撮影 → スコアが低い(△)ことを確認
4. わざとブレさせる・離す・近づける・照明を変えるなどして、スコアの変動を観察
5. 「MobileNetも使う」をONにして同じことを繰り返し、2方式のスコアを比較
6. しきい値スライダーを動かして「同じ物は通り、別の物は通らない」境界を探す

記録の目安: 同一物のスコアの最低値と、別物のスコアの最高値の間に隙間があれば、その方式は成立。

1. `https://mt44kr-eng.github.io/ofuro-sakana-zukan/ume/` を自分のLINE(Keepメモなど)に送る

   - 「0ばんめのさかな」→ 氷を3回タップ → 割れてドクターフィッシュが出るか

- 撮影画像は端末内でのみ処理。サーバー送信なし

# おふろ魚図鑑アプリ(仮称)

3〜5歳児の「お風呂渋り」を解決する商品「おふろ魚図鑑」の相棒PWA。
21日間、氷から出てきた魚フィギュアを風呂上がりに撮影すると正体(名前・豆知識)が明かされる図鑑アプリ。

## 現在の状態: ステップ1(カメラ+照合の技術検証スパイク)

最小構成のPWAで以下を検証する:

1. iPhoneでホーム画面追加した状態でカメラが起動・撮影できるか
2. 1対1照合(今日の魚が写っているか)が、子どものブレ写真でも成立するか

### 照合方式(2方式を並走して比較)

| 方式 | 実装 | 特徴 |
|------|------|------|
| 色ヒストグラム | `js/matcher-histogram.js`(依存ゼロ) | 画像中央のHSV色分布を比較。軽いが背景・照明の影響を受ける |
| MobileNet | `js/matcher-mobilenet.js`(TensorFlow.js CDN・初回約3MB) | 特徴ベクトルのコサイン類似度。ブレ・角度に強い。学習不要 |

どちらも「基準写真(最大数枚)との最大類似度 ≧ しきい値」で判定。しきい値はテスト画面のスライダーで調整できる。

## iPhone実機での検証手順

### A. カメラ+ホーム画面追加の検証(最重要)

1. iPhoneのSafariで公開URLを開く → カメラ許可を求められたら「許可」
2. プレビューが映り「📷 撮影する」で撮影できることを確認
3. 共有ボタン → 「ホーム画面に追加」
4. ホーム画面のアイコンから起動 → 画面右上が「ホーム画面起動: ✅」になっていることを確認
5. その状態でもう一度カメラ起動・撮影できることを確認
   (iOSのバージョンによっては起動のたびにカメラ許可を聞かれる。その頻度もメモしておく)

### B. 照合精度の検証(自宅にある物でOK)

フィギュアが未完成でも、色や形が異なる小物(ぬいぐるみ・おもちゃ等)2〜3個で代用できる。

1. 「基準を登録」モードで物Aを2〜3枚撮影(角度を変えて)
2. 「判定する」モードで **同じ物A** を撮影 → スコアが高い(○)ことを確認
3. **別の物B** を撮影 → スコアが低い(△)ことを確認
4. わざとブレさせる・離す・近づける・照明を変えるなどして、スコアの変動を観察
5. 「MobileNetも使う」をONにして同じことを繰り返し、2方式のスコアを比較
6. しきい値スライダーを動かして「同じ物は通り、別の物は通らない」境界を探す

記録の目安: 同一物のスコアの最低値と、別物のスコアの最高値の間に隙間があれば、その方式は成立。

```
index.html              照合テスト画面

sw.js                   Service Worker(ネットワーク優先)
css/style.css
js/app.js               画面制御
js/camera.js            カメラ起動・撮影
js/matcher-histogram.js 照合方式1: 色ヒストグラム
js/matcher-mobilenet.js 照合方式3: MobileNet(遅延読み込み)
icons/                  PWAアイコン(仮)
```

## 方針メモ

- 撮影画像は端末内でのみ処理。サーバー送信なし
- 判定は不正検知ではなく儀式の演出。物理的に子どもは「今日の魚」を持っているので、ゆるい判定+やさしいリトライ誘導でよい
- 否定語(「失敗」「ちがう」)はUIに使わない
